Overview of Mobile Development Options for the Google Maps API

Mano Marks, Google Geo APIs Team
December 2009

Contents

  1. Objective
  2. Introduction
  3. Google Maps API V3 in the Browser
  4. Google Maps API V3 in a Browser Embedded in a Native Application
  5. Native Google Maps APIs
  6. Static Maps API

Objective

Several approaches to creating applications using Google Maps on mobile devices exist. This article presents an overview of these options, and helps you choose which option works best for your application.

Introduction

Traditionally, Maps applications have been developed for viewing in web browsers running on desktop computers. While there are design considerations based on the different display sizes and browser capabilities used, the range of different devices and capabilities isn't nearly as great as that in mobile devices. In order to develop for mobile browsers, engineers have to carefully choose a development platform based on their target users. These constraints will likely start to melt away as devices standardize on HTML 5 based JavaScript capable browsers. However, in the meantime, Google provides you with options on different devices.

Google Maps API V3 in the Browser

The most capable phones — currently only Android devices and iPhones — can use Google Maps API directly in the browser. Version 3 of the Google Maps API was designed for use in these mobile browsers. This new API has low latency and provides you with a growing variety of features.



Advantages:

  • You allow users on any mobile device that supports HTML 5 and JavaScript to find your page and load it on their device.
  • You can use the same page for people searching the web on any device, including computers and mobile devices.
  • You can iterate quickly. You are not dependent on having device specific native code.
  • Users don't have to update their application when you make changes, they simply load the page directly and your changes are there.
  • When Google releases bug fixes or adds new features, they are automatically added to your web page.
  • Some phones, including Android and iPhone devices, have started supporting the W3C Geolocation in the browser, which allows JavaScript pages to utilize the current location of the device.
  • JavaScript API provides built-in support for many features that the existing native APIs do not, such as polylines and directions.

    • Disadvantages

    • On smart phones, particularly Android, iPhone, and Blackberry, some users rely on an App Store or Market to discover new content. Those users may not find your page without a native app.
    • You can't take advantage of special sensors in the loading device, such as the accelerometer and compass, unless they expose that through a JavaScript interface.

    Google Maps API V3 in a Browser Embedded in a Native Application

    Smart phones generally have objects that allow you to embed a web browser into your application. This capability allows you to load your page into an application running on the phone. For more information, consult the following tutorials on developing embedded V3 pages on mobile devices:

    • Loading a Maps API Site in a Native Androd Application
    • Developing Native iPhone Applications using V3.


    Advantages:

    • You can iterate without updates, and cross-platform support, just like loading the page directly in a browser.
    • Apple App Store/Android Market availability provides another channel for discovery.
    • Lightweight browser code may be easier to deal with than native APIs.
    • You can reuse the map portion of your application across different platforms, including the web.
    • You can take advantage of native data storage to store user data.

    Disdvantages:

    • Additional permission requirements on applications may scare off a few users.
    • Embedded browsers may be more limited than launchable browsers. Android, for instance, doesn't allow you to use Gears with the embedded browser, and only allows W3C Geolocation in version 2.0 or later.
    • You have to maintain different code bases for different platforms.

    Native Google Maps APIs

    Two platforms, the iPhone and Android, currently provide native API access to Google Maps. These APIs are limited to the operating system of the phone they were built for, and are maintained by those developers. New features in the native APIs are subject to the release cycles of those platforms. Currently, only the iPhone and Android platforms have native Google Maps APIs.

    Advantages:

    • Native access means only one code base per application. No need to separately maintain a web server just for the map.
    • Native APIs have better performance than loading a web page, though that gap is narrowing.
    • Tighter app integration with sensors on the device.

    Disadvantages:

    • Native APIs are developed at a slower pace, only changing when the platform provider releases a new version of the platform OS. That may mean a long wait to get new features in the API.
    • The Native APIs are very different on Android and the iPhone. Their interfaces are very different and they have different capabilities. Therefore, you have to maintain a separate architecture for iPhone and Android.
    • Neither has the full capabilities of the Google Maps API for JavaScript.
    • With only an application in the Apple App Store or Android Market, web discoverability is low.
    • You have to rely on users to update your application, and for updates to go through the App Store and Market approval process.

    Static Maps API

    Most mobile devices still do not support full, or any JavaScript in their browsers. And most devices don't provide access to a native mapping API. Google Maps provides a solution to this problem. The Static Maps API allows you to request an image using a URL.



    Advantages:

    • Very low bandwidth
    • Very low latency
    • Simple development
    • Widely supported

    Disadvantages:

    • It's only an image, not a slippy map. So it looks good but you can't interact with it.
    • Does not have the wide set of features of the JavaScript Maps API V3
