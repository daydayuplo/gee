How do I load the Google Maps APIs using a Maps API Premier client id?

Detailed information on loading the Google Maps APIs using a Maps API Premier client id is available in the Google Maps API Premier Developer's Guide.

Which APIs are included in Google Maps API Premier?

The following Google APIs are included in Google Maps API Premier. Applications that use these APIs with a Google Maps API Premier client id are covered by the Google Maps API Premier SLA, and are eligible for technical support:

  • Google Maps JavaScript API V3
  • Google Maps JavaScript API V2
  • Google Maps API for Flash
  • Google Static Maps API
  • Google Earth API
  • Google Geocoding API
  • Google Elevation API
  • Google Directions API

The following APIs are not covered by Google Maps API Premier. Applications that use these APIs must do so in compliance with the published Terms of Service for these APIs, and are not covered by the Google Maps API Premier SLA, or eligible for technical support at this time:

  • Google Maps Data API
  • Google Places API
  • Google AJAX Search API (including Local Search)
  • Android Maps External Library
  • Apple iPhone MapKit framework
How is Google Maps API Premier usage tracked and reported?

For Maps API Premier licenses that are limited to a specific number of page views, a page view is defined as:

  • A single load of the Google Maps JavaScript API into a page. The JavaScript API is reloaded every time a page that uses the API is reloaded. User interactions with the map (eg. panning, zooming, changing map types) do not generate more page views. Note however that a page view is generated if the API is loaded into the page even if the API is not then used to display a map. We therefore recommend against including the <script> tag that loads the API in page templates, and recommend loading the API asynchronously on pages where maps are only displayed in response to optional user activity.
  • A single load of the Google Maps API for Flash. The Maps API for Flash is loaded every time a Flash application (SWF file) that requires the Maps API for Flash is initialized.
  • A single request for a Google Static Maps API image.

Maps API Premier customers can track their page view usage on the the Google Enterprise Customer and Partner Site at:

http://www.google.com/enterprise/portal

How do I contact technical support for Google Maps API Premier?

Maps API Premier customers can open a new support case with Google Enterprise Support using the Google Enterprise Customer and Partner Site at:

http://www.google.com/enterprise/portal

Use the Contact Us link on the left hand side once logged in to access the form used to open a new support case.

How do I reset login credentials for the Google Enterprise Customer and Partner Site?

The Google Enterprise Customer and Partner Site is accesible to all Google Maps API Premier customers by using your Maps API Premier client id as the Username, and the password sent to you when your Maps API Premier license was issued. If you do not have access to this password you can use the Forgot Your Password? link on the login page to have a new password sent to the email address associated with your account. If this address is not correct please email enterprise-operations@google.com to request that it be changed.

How do I access the Google Maps APIs over HTTPS (SSL) using a Maps API Premier client id?

To access the Google Maps APIs over HTTPS you must first notify the Maps API Premier support team of the HTTPS sites on which you wish to use the Maps API. Once the support team have confirmed that your Maps API Premier client id has been enabled for use on these sites, you can load the Maps APIs over https as documented in the Google Maps API Premier Developers Guide.

Can the Google Maps API for Flash be accessed over HTTPS (SSL) using a Maps API Premier client id?

It is now possible to load the Maps API for Flash over SSL using a Google Maps API Premier client id. To do so you must build your Flash application using version 1.18 or later of the Maps API for Flash and set the ssl property to true as documented in the Google Maps API Premier Developers Guide.

How do I load the Google Maps JavaScript API over HTTPS (SSL) when using the Common Loader base_domain parameter?

It is not possible to load the Maps JavaScript API over SSL using the common loader with the base_domain parameter set. However use of the base_domain parameter does not affect application performance because API requests are always handled by the optimal Maps server for a particular user given their location regardless of the domain from which the API is loaded.

The only impact that the base_domain parameter has when the API is loaded over http is to implicitly bias geocoding results to that country. You can force the same behaviour in your application using GClientGeocoder.setBaseCountryCode().

Which features of the Google Maps JavaScript API v2 are available over HTTPS (SSL)?

All features of the Google Maps JavaScript API v2 are available over HTTPS with the following exceptions:

  • MapMaker map types.
  • Sky, Moon, and Mars map types.

What are the usage limits for the Google Maps API Web Services?

Maps API Premier customers are provided with higher usage limits for the Google Maps API Web Services:

  • For the Geocoding API: 100,000 requests per day.
  • For the Directions API: 100,000 requests per day and a maximum of 23 waypoints per request.
  • For the Elevation API: 100,000 requests per day, a maximum of 512 points per request and a maximum of 1,000,000 points per day.

These limits apply to each Maps API Premier client id and must be shared across all applications and IP addresses using a given client id. There is also a rate limit of 10 requests per second that applies to all Google Maps API Web Services.

If you need higher usage limits, please contact your Maps API Premier Account Manager for purchase information.

What are the usage limits for Geocoding requests made within the Maps JavaScript API V2 and the Maps API for Flash?

Geocoding requests made by applications using the JavaScript API V2 or the Maps API for Flash are subject to the limit of 15,000 requests per IP address per day outlined in the Maps API FAQ. By limiting client application geocode usage per IP address in this way, the number of geocoding requests your application can generate increases in line with your user base.

The JavaScript API V3 uses a different approach to limit the requests volume your applications can generate. A larger daily allocation is granted to each user session, regardless of how many users share a given IP address.

There are much higher daily limits for the requests described as page views.

Can I use the GoogleBar or AJAX Search API on a Maps API Premier site?

The GoogleBar is simply a convenient way of embedding a Local Search control that uses the AJAX Search API. The AJAX Search API is distinct from the Maps API with it's own Terms of Use and is therefore not incldued in Maps API Premier. Consequently any Maps API Premier application that wishes to use the AJAX Search API or the GoogleBar must comply with the AJAX Search API Terms of Use, including complying with any clauses that are more restrictive than the Maps API Premier Terms. This means for example that it is not permitted to use the GoogleBar or Local Search API on a site that is not generally accessible to users without charge.

How do I disable the display of advertising when using the GoogleBar on a Maps API Premier site?

The GoogleBar embeds a Local Search control provided by the AJAX Search API in your map. The AJAX Search API is distinct from the Maps API with it's own Terms of Use and is therefore not included in Maps API Premier. As a result it is not possible to disable the display of advertising when using the GoogleBar. If your application complies with the AJAX Search API Terms of Use you can add business search without using the GoogleBar or Local Search control as illustrated in this example. However Google reserves the right to deliver advertising through the AJAX Search API at any time in the future.

How do I render KML files that are hosted on intranet sites on a map?

The GGeoXML class that generates KML overlays in the Google Maps API uses a Google hosted service to retrieve and parse KML files for rendering. Consequently it is not possible to display KML files that are not hosted at a URL that is available publicly accessible, or that require authentication to access.

If you need to develop applications that use KML files hosted on intranet sites we recommend that you use the Google Earth API which supports client side parsing and rendering of KML.

Can I sign URLs using JavaScript?

We strongly recommend against signing URLs using JavaScript as this would expose your cryptographic key to end users. Therefore signatures should only be generated by server side components.

Why am I receiving a HTTP 403 Forbidden response to my Maps API web service requests?

A HTTP 403 response indicates a permission issue, likely because the signature could not be verified for this request. This could be because:

  1. A signature has been specified but is incorrect for this request.
  2. The request specifies a Maps API Premier client id but does not specify a signature, and the web service being called requires that all requests made using a client id include a valid signature.
  3. A signature has been specified but the associated Maps API Premier client id has not been specified.
