Java Class Library

The Search Framework Java Class Library provides classes for storing request and response data in the generic Common Search Format.

To use the Search Framework Java Class Library, you need to include this line in your code to refer to the SearchAPI.jar file:

import com.google.earth.search.api.*;

Be sure the SearchAPI.jar file is in your CLASSPATH.

The following table summarizes the classes and functions included in this library.

Class Description
GeomMetaData A simple data structure for holding per-geometry metadata such as:
name - a label for the geometry snippet - a short piece of text to associate with the geometry description - text (possibly lengthy) to associate with the geometry
KmlPluginPreferences Abstract class for pluging preferences for general purpose KML returned by the various Google Earth Search plugins.
KmlProperties A container of KML property keys for use within the search framework.
KmlSearchPlugin
This abstraction provides Kml style search response using KmlPluginPreferences.
PluginPreferences Abstract class for preferences for the KML returned by the various plugins.
SearchContainer The object that stores a single search request issued by any supported interface and stores any number of search responses to this request.
SearchPlugin Interface for the search plug-in application. Provides the init() and doSearch() methods, which the plug-in must implement.
SearchContainer Stores one search request and any number of search responses to this request.
SearchRequest Object that encapsulates the key/value pairs that make up the search query. This class provides the getService() method, which returns the name of the service that should handle this request.
SearchResponse Object that encapsulates one or more results sets created by a single search plug-in application.
StyleData Class for representing styles associated with geometry. Fields are

  • balloonStyle - Hash map includes:
    • bgColor
    • textColor
    • text
  • iconStyle - Hash map includes:
    • color
    • scale
    • href
  • lineStyle - Hash map includes:
    • color
    • width
  • polyStyle - Hash map includes:
    • color
    • colorMode
    • fill
    • outline

Currently, styles are recognized only by Google Earth plug-ins. These styles correspond to the KML styles of the same names. (For more information, see the KML Reference.)
